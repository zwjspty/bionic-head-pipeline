# Task 11.4 High-Priority Outbound Queue Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ensure interruption control events such as `server.playback.stop` and `server.turn.cancelled` are sent before queued normal stream payloads, while stale normal events are dropped before network send.

**Architecture:** Add a connection-local priority outbound queue. Normal JSON/binary-pair messages are enqueued and return after queueing, so `TurnHandle.emit_if_current()` no longer holds its lock while awaiting WebSocket network I/O. High-priority terminal/control messages wait until sent, and the queue re-checks `generation_epoch` immediately before sending normal messages.

**Tech Stack:** Python 3.11, asyncio `PriorityQueue`, FastAPI WebSocket send APIs, existing `EventEnvelope` protocol models, pytest.

## Global Constraints

- Work on branch `task11-cancel-safe-sidecar`.
- Do not change WebSocket event schema.
- Do not change sidecar protocol.
- Preserve JSON + binary pairing for `server.tts.audio`.
- `server.playback.stop`, `server.turn.cancelled`, and `server.pipeline.error` must be high priority.
- Normal queued turn-scoped messages with stale `generation_epoch` must be dropped before send.

- [x] **Step 3: Implement queue**

Use an `asyncio.PriorityQueue` whose sort key is `(priority, sequence)`. Queue entries contain:

```text
priority
sequence
envelope
binary optional
completion future optional
```

Start the sender lazily after enqueue. Enqueue before starting the sender so multiple sends issued in one event-loop tick can be ordered by priority.

- [x] **Step 4: Add stale-drop check**

Before sending a normal message, if `envelope.turn_id is not None` and `envelope.generation_epoch != self.generation_epoch`, drop it and increment `_outbound_stale_drop_count`. High-priority messages bypass this drop.
