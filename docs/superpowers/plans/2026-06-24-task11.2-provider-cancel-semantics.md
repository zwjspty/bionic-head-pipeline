# Task 11.2 Provider Cancel Semantics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make WebSocket turn cancellation best-effort notify every provider without delaying `server.playback.stop`.

**Architecture:** Keep the existing adapter `cancel(turn_id)` interface. Add a registry-level `cancel_turn(...)` fanout that records per-provider results, expose it through `AppContainer`, and have `StreamConnection` schedule that work in the background during turn cancellation.

**Tech Stack:** Python 3.11, asyncio tasks, existing adapter wrappers, pytest.

## Global Constraints

- Work on branch `task11-cancel-safe-sidecar`.
- Do not change WebSocket event schema in this task.
- Do not change sidecar protocol.
- Do not make provider cancellation block `server.playback.stop`.
- Provider cancel failures must be captured as results, not surfaced to the client.
- Keep existing adapter `cancel(turn_id)` compatibility.
