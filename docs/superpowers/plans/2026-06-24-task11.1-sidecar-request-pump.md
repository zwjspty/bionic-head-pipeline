# Task 11.1 Sidecar Request Pump Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `EmoTalkSidecarAudio2FaceAdapter` safe when caller cancellation happens after a sidecar request has been written but before its response arrives.

**Architecture:** Keep the existing sidecar binary protocol and process lifecycle. Add a request pump inside `src/bionic_head/adapters/emotalk_sidecar.py` that serializes stdin/stdout transactions independently of caller tasks, drains cancelled responses, and discards them before accepting the next request.

**Tech Stack:** Python 3.11, asyncio subprocess stdin/stdout, pytest, existing `SidecarRequest` / `SidecarResponse`.

## Global Constraints

- Work on branch `task11-cancel-safe-sidecar`.
- Do not change `emotalk-sidecar-v1` protocol.
- Do not change worker code.
- Do not implement OrderedFaceSequencer or outbound queue in this task.
- Preserve existing timeout behavior: provider timeout closes the sidecar process.
- Preserve protocol-error behavior: invalid response closes the sidecar process.
- Automated tests must use fake sidecar scripts and must not require real EmoTalk.

- [ ] **Step 3: Implement pump**

Inside `emotalk_sidecar.py`:

```text
SidecarTransaction:
  request
  metrics
  future

SidecarRequestPump:
  queue
  task
  close()
  submit()
  run loop:
    get transaction
    run adapter._transact_uncancellable()
    if future cancelled: discard response and increment counter
    else: set result / set exception
```

Change `drive()` / `prewarm()` to call pump through `asyncio.wait_for(asyncio.shield(future), timeout=...)`.
