# Task 11.5 Interrupt Stress Benchmark Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add repeatable interrupt stress evidence that no old-generation face/UE5 events leak after playback stop.

**Architecture:** Add a small external WebSocket stress script for manual benchmark runs, plus a mock WebSocket integration test that exercises repeated barge-in interruption in-process. The script records event counts, playback stop count, stale face leaks, and sequence validity.

**Tech Stack:** Python 3.11, websockets, FastAPI TestClient, existing stream protocol helpers, pytest.

## Global Constraints

- Work on branch `task11-cancel-safe-sidecar`.
- Do not require real ASR / LLM / TTS / EmoTalk for automated tests.
- Manual script may require a running server.
- Default stress target is 50 interrupts for acceptance evidence.
- Any `server.face.frames` or `server.ue5.frames` event with older `generation_epoch` than the latest observed generation is counted as an old-turn face leak.
