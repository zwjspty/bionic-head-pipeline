# Task 11.3 Ordered Face Sequencer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow Audio2Face segment inference to finish out of order while guaranteeing face postprocessing and outbound UE5/segment events are emitted in `segment_index` order.

**Architecture:** Split stream face work into two phases. Background tasks only run `audio2face.drive(...)` and return raw `FaceArtifact`. The main stream orchestrator drains completed face tasks through an ordered sequencer that applies stitch/eye continuity, formats UE5, and emits `server.face.frames`, `server.ue5.frames`, and `server.segment.ready` only when all earlier segments have been emitted.

**Tech Stack:** Python 3.11, asyncio tasks, existing `FaceSegmentStitcher`, `EyeContinuityProcessor`, pytest.
