# Task 11.1 Sidecar Request Pump Design

## Goal

Task 11.1 makes the EmoTalk sidecar adapter cancellation-safe. Cancelling a caller while a sidecar request is already in flight must not leave the old response in stdout for the next request to accidentally read.

## Problem

Current flow:

```text
drive()
  -> acquire _request_lock
  -> write request
  -> await response
```

If the awaiting coroutine is cancelled after the request was written, the worker can still finish and write the old response. The adapter no longer awaits that response, so the next request may read the old response and fail with session/turn/generation mismatch.

## Design

Add a sidecar request pump owned by `EmoTalkSidecarAudio2FaceAdapter`:

```text
SidecarRequestPump
  owns stdin/stdout transactions
  serializes requests
  never lets caller cancellation cancel the underlying read
  drains old responses and discards them when caller future is cancelled
```

Public adapter behavior:

```text
drive()
  -> build SidecarRequest
  -> submit request to pump
  -> await caller future

if caller is cancelled:
  -> future is cancelled
  -> pump continues reading old response
  -> pump discards response
  -> sidecar stdout remains aligned
  -> next request can reuse same warm process
```

Timeout behavior remains conservative:

```text
timeout while caller is waiting
  -> close sidecar process
  -> pump stops
  -> next request starts fresh process
```

Protocol validation errors also close the sidecar, as before.

## Metrics

Add adapter counters:

```text
sidecar_request_cancelled_count
sidecar_response_discarded_count
sidecar_protocol_reset_count
sidecar_process_restart_count
```

Persist per-request metrics where useful:

```text
sidecar_inflight_count
sidecar_request_cancelled_count
sidecar_response_discarded_count
```

## Acceptance

```text
1. A request already written to fake sidecar can be cancelled.
2. Fake sidecar later returns the old response.
3. Pump drains and discards old response.
4. Immediate next request succeeds.
5. process_start_count remains 1.
6. No protocol mismatch is raised.
7. Existing timeout/protocol-error tests still pass.
```

## Out of Scope

```text
No OrderedFaceSequencer.
No StreamConnection outbound queue.
No provider cancel fanout.
No true model interruption.
No sidecar protocol change.
```
